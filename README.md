# Sportrack — Мобільний додаток для відстеження тренувань

Проєкт бакалаврської роботи. Система для планування тренувань, контролю прогресу та аналітики фізичних показників.

## Основні функції:
- Розрахунок прогресії ваги та повторень.
- Створення власних тренувальних планів та щоденник тренувань.
- Аналітика: графіки силових показників, ваги та об'ємів тіла.
- Гнучкі фільтри вправ за м'язовими групами та обладнанням.

## Структура проєкту:
Додаток побудований за архітектурою MVVM з використанням сучасних інструментів Android-розробки:
- `data/` — Локальне сховище даних на базі бібліотеки Room. Включає моделі (`Exercise`, `Workout` тощо) та Data Access Objects (`DAO`).
- `ui/` — Користувацький інтерфейс, побудований на Jetpack Compose. Поділений на функціональні модулі (`catalog`, `progress`, `profile` тощо) зі своїми ViewModel.

## Правила документування коду (Contribution Guidelines)

У проєкті Sportrack використовується стандарт **KDoc** та генератор **Dokka**. Усі розробники зобов'язані дотримуватись наступних правил:

1. **Публічні інтерфейси:** Усі класи (крім внутрішніх Compose-компонентів), публічні методи ViewModel та DAO повинні мати KDoc коментар.
2. **Формат:** * Починається з `/**` та закінчується `*/`.
   * Перший рядок — короткий опис (Summary).
   * Далі — детальний опис (за потреби).
   * Теги: `@param` (опис параметрів), `@return` (що повертає), `@throws` (можливі виключення).
3. **UI Компоненти:** Для складних Jetpack Compose функцій обов'язково описувати стан (`State`), який вони приймають.
4. **Багатомовність:** Основна документація в коді ведеться англійською мовою (для універсальності), архітектурні документи дублюються українською (напр., `README.uk.md`, `architecture.uk.md`).
