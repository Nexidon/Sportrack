# Архітектура Sportrack (Business Logic & Components)

## 1. Загальний підхід
Проєкт побудований за принципами Clean Architecture та шаблоном MVVM (Model-View-ViewModel).

## 2. Взаємодія компонентів
* **UI Layer (Jetpack Compose):** Відображає дані та передає події користувача у ViewModel. Усі компоненти "дурні" (stateless).
* **Presentation Layer (ViewModel):** Обробляє бізнес-логіку. Спілкується з репозиторіями, отримує дані через Kotlin Flows та конвертує їх у UI State.
* **Data Layer (Room DAO):** Відповідає за кешування та збереження локальних даних на пристрої.

## 3. Складні алгоритми (Алгоритм прогресії)
Модуль розрахунку прогресії використовує формулу Бжицькі (Brzycki) для розрахунку одноповторного максимуму (1RM), що дозволяє коректно малювати графіки прогресу навіть якщо користувач виконував вправи з різною вагою та кількістю повторень.
